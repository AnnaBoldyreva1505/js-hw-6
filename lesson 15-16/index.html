node js

node js - не просто программа, а полноценная среда разработки. Позволяет не только писать бэкенд, но и исполнять js код. Не обязательно в консоли браузера.

node version manager(nvm) - софт позволяет переключаться между версиями node js, если на проектах разные версии

node packege manager - помогает установкой пакетов для нод js (плагины - babel, библиотеки - например реакт)

yarn - устанавливается отдельно, тоже установщик пакетов

НЕЛЬЗЯ ПУШИТЬ ПАПКУ node_modules на github. Для того, чтобы исключить пуш туда, нужно создать файл git ignore и в него прописать ограничения
.gitignore / в файле написать /node_modules

webpack

Структура - которую прописывают, создают руками, через подключение библиотеки. вебпак лежит в основе библиотке реакт. он позволяет окнтролировать работу, загрузку всех файлов и библиотек из нод модульс, подгружать коды и скрипты тогда, когда нужно а не сразу. он будет подключать только один файл js, в него будут подключаться все остальные через импорт. Важно не забывать делать экспорт, если это не файл а отдельная функция

данные делятся на реляционные (SQL - как пример)

JSON

JSON создан от объекта. его отличие малый вес, к примеру 4 тома войны и мир можно вместить в 5Мб У него есть 2 метода: parse() - парсит, разбивает.Используется для того чтобы из JSON сделать то, что должно быть stringify() - приводит к строке. приводик к json

local storige

Находится во вкладке Application в дев тулз Storage - LocalStorage кукис - хранят данные, лучше не давать на них разрешения, так как могут утянуть данные, например и локал сторедж (данные карт, личные данные)

localStorage - объект, с кучей методов: длина, очистка (clear), getItem, key, length, removeItem, setItem - запись в локал сторидж
